SWIFT version history

Version-to-come

The version-to-come may or will include:

Version 0.5.0

4users

  • Most importantly: a completely new challenge is integrated: efeLang!

  • During the translation round, the shortest translation times will be shown: the personal record and the overall record. This is intended to further motivate the player to be as quickly as possible.

  • A progression bar is shown, allowing the player to easily jump back to previous rounds for review and/or corrections, and for an enhanced sense of overview for the player. (However, the rounds do not yet automatically rewind when a previous round is re-edited).

4developers

  • The introduction of efeLang is not so easy as it may seem. The previous challenge was still depended on some non-Scala code (Clean). EfeLang is completely written in Scala. Moreover, this has been done in an abstract, generic way. The latter allows other future challenges, with new CTL language and reasoning problems, to be integrated far more easily (I hope).

  • Moreover, the code is already partially prepared for allowing more than one challenge in the SWiFT game. Considerable refactoring has taken place, among other things, introducing some parameterized types to make it easier to add new challenges with improved reuse.

Version 0.4.0

August 8, 2013

4users

  • Stopwatch running during translation round to stimulate making the duration as short as possible.

  • When clicking “study chosen consti”, a new browser tab opens to study consti, or if it is already open, it will switch to that tab.

  • Browser will jump back to last scrolling position in the constitution being studied if a player returns to the study consti page.

4developers

  • Version number displayed in running SWiFT application in default.html now read from central location.

  • Version number now stored in a separate file in the root of the webapp (where pom.xml is).

  • SWiFT backup system prepared, but not yet fully tested because still waiting for a back-up server. See pom.xml.

  • Added new documentation of source code

Version 0.3.2

June 9, 2013

4users

  • Correct version number now also shows in web application.

  • Bugfix fluency game: names with hyphens and special characters are removed (these will lead to problems if they occur in the natural language text to translate).

4developers

  • H2 database is made accessible through Web interface.

Version 0.3.1

4users

4developers

Version 0.3.0.1

(Deviation from version naming protocol used in SWiFT here: the 4th number should not be used.)

4developers

  • PersistentDatastructureUpgrader added. This piece of code applies the right upgrades to the persistently stored data, if a server is updated with a new version of SWIFT, but still has data stored in an older format. The upgrade-algorithms belonging to specific versions can simply be added to the list of upgraders, and will be applied when necessary.
